# Tests (root)

Diretório de testes adicionais fora do app (contratos/e2e/performance).

- `tests/contract` – validações de contrato/API (padrão a definir).
- `tests/e2e` – espaço para E2E (definir ferramenta: Playwright/Cypress).
- `tests/performance` – testes/perf benchmarks se necessário.

Comandos globais de teste do projeto: `pnpm test` (Jest). Ajuste ou adicione scripts específicos para estes diretórios conforme a ferramenta escolhida.
